# CHAT-ROADMAP.md — AADS 채팅 시스템 진화 로드맵

_최종 업데이트: 2026-03-27_

---

## 현재 위치

Phase 2 운영 중 (AADS-190 완료). 원격 쓰기/실행 + 서브에이전트 + 무한 대화 구현 완료.
다음 목표: 에이전트 투명성 강화 + CEO 의사결정 지원 UI.

---

## Phase 1 — 즉시 개선 (1~2주)

### 목표
현재 구현된 백엔드 기능을 UI에서 충분히 활용하지 못하는 부분을 보완한다.

### 핵심 기능

#### 1-A. 도구 실행 진행률 표시
- 현황: 도구 호출 횟수(N회)만 버블에 표시
- 목표: 진행 중인 도구명 + 예상 남은 시간 인디케이터
- 구현: SSE tool_use 이벤트에 포함된 tool_name 실시간 렌더링 개선
- 파일: `src/app/chat/page.tsx` (tool_use 렌더링 섹션)

#### 1-B. 비용/턴 카운터 상시 표시
- 현황: SSE done 이벤트로 session_cost / session_turns 수신하나 UI 미표시
- 목표: 입력창 우측 상단 `$0.12 | 7턴` 형태 상시 표시
- 구현: StreamState.sessionCost / sessionTurns → 입력창 옆 뱃지
- 파일: `src/app/chat/page.tsx`, `src/hooks/useChatSSE.ts`

#### 1-C. Yellow 도구 경고 UI 개선
- 현황: yellow_limit SSE 이벤트 수신하나 텍스트 알림만
- 목표: [계속 실행] / [중단] 버튼이 있는 모달 또는 인라인 배너
- 구현: yellow_limit 이벤트 처리 + 승인 흐름 (POST /{session_id}/stop 연동)
- 파일: `src/hooks/useChatSSE.ts`, `src/app/chat/page.tsx`

#### 1-D. 참고자료 통합 UI
- 현황: CitationCard / SourceCard 컴포넌트 존재하나 메시지 버블 연동 미흡
- 목표: 검색 도구 결과 → 메시지 하단 자동 참고자료 섹션 표시
- 구현: tool_result 중 search/research 타입 → SourceCard 자동 추출

#### 1-E. 인라인 인용 마커
- 목표: AI 응답 본문에서 `[1]`, `[2]` 형태 인용 번호 → 참고자료 카드 연결
- 구현: MarkdownRenderer에 인용 번호 파싱 + CitationCard 링크

### 예상 효과
- CEO가 작업 비용을 실시간 인지 → 불필요한 긴 작업 조기 중단 가능
- 도구 진행 상황 가시화 → 대기 시 불안감 감소
- 검색 기반 응답의 출처 투명성 향상

## Phase 2 — 단기 개선 (2~4주)

### 목표
에이전트 작업 투명성을 높이고 CEO가 진행 상황을 능동적으로 모니터링할 수 있게 한다.

### 핵심 기능

#### 2-A. 에이전트 투명성 패널
- 목표: 현재 실행 중인 에이전트(Agent SDK / 서브에이전트)의 내부 상태 표시
  - 어떤 도구를 호출했는지
  - 각 도구 호출의 결과 요약
  - 총 소요 시간 / 비용
- 구현: 아티팩트 패널 내 새 탭 "에이전트" 추가
- 파일: `src/app/chat/ChatArtifactPanel.tsx`, 신규 `ArtifactAgentTrace.tsx`

#### 2-B. 실시간 작업 트래커
- 목표: 진행 중인 Directive / 장기 작업의 상태를 채팅 사이드바에서 확인
- 구현: ArtifactTaskMonitor 컴포넌트 사이드바 통합 (현재 분리된 컴포넌트)
- 파일: `src/components/chat/ArtifactTaskMonitor.tsx`, `src/app/chat/ChatSidebar.tsx`

#### 2-C. 메모리 시각화
- 목표: MemoryContextBar를 확장하여 5섹션 메모리 내용을 직접 확인/편집
- 구현: GET /chat/sessions/{id}/memory-context 결과를 확장 패널로 표시
- 파일: `src/components/chat/MemoryContextBar.tsx`

#### 2-D. 서브에이전트 병렬 작업 표시
- 목표: spawn_parallel_subagents 실행 시 병렬 진행 상황을 시각적으로 표시
- 구현: 병렬 서브에이전트별 독립 진행 바 + 완료 순서 표시
- 파일: 신규 `SubagentProgressCard.tsx`

#### 2-E. 세션 검색 강화
- 현황: GET /chat/messages/search 구현됨, UI 미구현
- 목표: 사이드바 내 검색창 → 모든 세션 전문 검색
- 구현: 검색 결과 세션 하이라이트 + 해당 메시지로 점프

### 예상 효과
- Agent SDK 실행 중 CEO가 중간에 끼어들거나 방향을 수정 가능
- 장기 작업(30분+) 중에도 진행 상황을 신뢰할 수 있음
- 메모리 시스템이 실제로 작동하는지 CEO가 직접 확인 가능

## Phase 3 — 중기 개선 (1~2개월)

### 목표
멀티모달 확장 + 다른 대시보드 기능과 채팅의 통합.

### 핵심 기능

#### 3-A. 멀티모달 입출력 강화
- 스크린샷 자동 첨부: CEO 화면 캡처 → 즉시 채팅에 붙여넣기
- 차트/그래프 자동 생성: AI 응답 내 데이터 → 아티팩트 chart 탭 자동 등록
- PDF 인라인 뷰어: PDF 첨부 시 내용 미리보기 패널

#### 3-B. 음성 인터페이스
- 음성 입력: Web Speech API → 텍스트 변환 후 전송
- 음성 출력: AI 응답 TTS (주요 요약 문장만)
- 모바일 최적화: 음성 버튼을 ActionChips에 추가

#### 3-C. Ops 대시보드 통합
- 목표: 채팅에서 Ops 데이터(헬스체크, 비용, 작업 현황)를 인라인으로 확인
- 구현: 채팅 우측 패널 "dashboard" 탭에 서비스별 상태 위젯 통합
- 파일: `src/components/chat/ArtifactDashboard.tsx` 확장

#### 3-D. 컨텍스트 히스토리 탐색
- 목표: 구조화 요약(compaction) 이력을 타임라인으로 확인
- 구현: 세션 내 컴팩션 포인트 마커 + 이전 요약 조회 UI

#### 3-E. Prompt Caching 히트율 표시
- 목표: 세션별 캐시 히트율 / 절감 비용 표시
- 구현: done 이벤트의 cache_read_input_tokens 활용

### 예상 효과
- 모바일에서도 완전한 채팅 운영 가능
- 채팅창 하나로 전체 시스템 상태 파악 + 지시 가능
- 비용 효율 가시화로 모델 선택 최적화

## Phase 4 — 장기 개선 (2~6개월)

### 목표
채팅을 단순 인터페이스가 아닌 CEO 자율 의사결정 지원 시스템으로 격상한다.

### 핵심 기능

#### 4-A. 자율 에이전트 오케스트레이션 UI
- 목표: 여러 서브에이전트가 병렬로 작업하는 오케스트레이션을 시각적으로 관리
- 기능:
  - 에이전트 의존성 그래프 표시 (어떤 에이전트가 어떤 에이전트를 기다리는지)
  - 개별 에이전트 일시정지 / 재개 / 재지시
  - 에이전트 간 결과 교환 로그 확인

#### 4-B. CEO 의사결정 지원 시스템
- 목표: AI가 CEO에게 "지금 이 결정이 필요합니다"를 능동적으로 알림
- 기능:
  - 승인 대기 중인 항목 채팅 상단 배너 표시
  - 결정 옵션(A/B/C)을 카드 형태로 제시
  - CEO 결정 이력 + 결과 추적

#### 4-C. 적응형 컨텍스트 압축
- 목표: Observation Window를 토큰 사용량에 따라 10→5→3으로 동적 조정
- 구현: context_compressor에 adaptive mode 추가

#### 4-D. RAG 기반 도구 결과 검색
- 목표: 마스킹된 이전 도구 결과를 벡터DB에 저장, 필요 시 재검색
- 구현: 도구 결과 임베딩 → pgvector 저장 → 세션 내 유사 결과 자동 주입
- 효과: compaction 이후에도 이전 도구 결과를 AI가 활용 가능

#### 4-E. 멀티 CEO 협업 모드
- 목표: 동일 세션에 여러 사용자가 참여하여 AI와 협업
- 구현: WebSocket 기반 실시간 공유 세션 + 발화자 구분 표시

### 예상 효과
- AADS가 "채팅창"에서 "자율 AI 운영 통제 센터"로 격상
- CEO가 직접 코드를 보지 않아도 6개 서비스 전체를 채팅으로 운영
- AI가 능동적으로 CEO에게 의사결정을 요청하는 역전된 상호작용
